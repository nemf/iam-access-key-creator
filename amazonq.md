# プロジェクトルール

## 応答ルール

- 必ず日本語で返答すること
- コード内のコメントや docstring も日本語で記述すること
- 技術用語は英語のままでよいが、説明文は日本語にすること
- 簡潔かつ実用的な回答を心がけること

## 概要

Strands Agents SDK を使用した AI エージェントプロジェクト。

## 使用ライブラリ

- **strands-agents**: エージェントのコアライブラリ
- **strands-agents-tools**: 組み込みツール群
- **pytest**: テストフレームワーク
- **pytest-asyncio**: 非同期テスト対応
- **ruff**: リンター・フォーマッター

## 開発フロー

- コードを書いたら、必ず対応するテストコードを作成する
- テストコード作成後、必ずテストを実行して全件パスを確認する
- Python の仮想環境は venv を使用する（`.venv/` ディレクトリ）
- パッケージインストールは `.venv/bin/pip` を使用する
- テスト実行は `.venv/bin/pytest` を使用する
- ソースを作成・編集したら、`README.md` を作成・更新する
- 既存コードを変更する場合は、変更前に影響範囲を確認する

## コーディング規約

- Python 3.12 以上を使用する
- 型ヒントを必ず付与する
- フォーマットは ruff format に従う
- リントは ruff check に従う
- docstring は Google スタイルで記述する
- 関数・変数名は snake_case、クラス名は PascalCase を使用する
- 定数は UPPER_SNAKE_CASE を使用する
- ツール定義には `@tool` デコレータを使用する
- エージェント生成時はモデルとツールを明示的に指定する
- モデルは Claude Opus 4.6 (`us.anthropic.claude-opus-4-6-v1`) を使用する
- マジックナンバーは定数として定義する

## エラーハンドリング

- 外部 API 呼び出しには適切な例外処理を実装する
- 例外は具体的な型でキャッチし、裸の `except` は使わない
- ユーザー向けのエラーメッセージは日本語で記述する
- ログ出力には `logging` モジュールを使用する

## セキュリティ

- 環境変数は `.env` で管理し、リポジトリにコミットしない
- シークレットやクレデンシャルをコードにハードコードしない
- ユーザー入力は必ずバリデーションする
- `.gitignore` に `.env`、`.venv/`、`__pycache__/` を含める

## テスト方針

- テストは `tests/` ディレクトリに配置する
- テストファイル名は `test_` プレフィックスを付ける
- ユニットテストでは外部 API 呼び出しを必ずモックする
- カバレッジ 80% 以上を目標とする
- `pytest` で実行する
- 正常系・異常系の両方をテストする

## Git 運用

- コミットメッセージは日本語で簡潔に記述する
- 機密情報を含むファイルはコミットしない

## ディレクトリ構成

```
.venv/            # Python 仮想環境
src/
  agents/         # エージェント定義
  tools/          # カスタムツール定義
tests/
  unit/           # ユニットテスト
  integration/    # 統合テスト
```
